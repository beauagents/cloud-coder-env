# Contributing

Thanks for contributing.

## For Everyone

- Keep changes small, focused, and reversible.
- Prefer clear commit messages and short PRs.
- Update docs when behavior or workflow changes.
- Do not commit secrets, credentials, or generated artifacts.

## For AI Coding Agents

- Read `AGENTS.md` before editing.
- Follow existing file/layout conventions.
- Avoid unrelated refactors.
- Run relevant lint/test commands before proposing changes.
- If no tests exist, document what was manually verified.

## Development Expectations

1. Create or update code in predictable directories (for example `src/`, `tests/`).
2. Keep formatting aligned with `.editorconfig`.
3. Ensure CI remains green.

## Pull Requests

- Use the PR template.
- Include:
  - What changed
  - Why it changed
  - Validation performed
  - Any follow-up work
